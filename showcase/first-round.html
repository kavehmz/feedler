the moment worth showing

It caught a bug in its own work — and fixed it

self-correction

“Reset everything” didn’t actually reset everything.

During the final checks, the AI ran the documented “wipe it all” command — docker compose down -v — and noticed the data survived. The reader’s database was living in a plain folder that command can’t remove. Because the spec promises that command is the clean-reset path, the AI moved the database into a proper managed volume, ran the reset again, and watched it truly start from zero. Nobody asked it to look.
